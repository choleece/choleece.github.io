小程序TabView

Veröffentlicht am 2018-01-16

下面我们将利用小程序的scroll-view & swipper制作tab-view

建立wxml

整个页面由顶部的标题栏+可滑动的body组成

<view class="stv-container">
  <view class="tab-bar">
    <view wx:for="{{titles}}" wx:key="unique" data-index="{{index}}" bindtap="handlerTabTap" class="tab {{activeIndex==index?'tab-active':''}}" style="width: {{stv.windowWidth/titles.length}}px">
        <text>{{item}}</text>
    </view>
    <view style="width: {{stv.lineWidth}}px; left: {{stv.offset/titles.length}}px" class="under-line {{!stv.tStart? 'withAnimate': ''}}"></view>
  </view>
  <view style="padding: 15px; height: 100%; margin-top: 30px; background-color: #fafafa">
    <swiper duration="500" bindchange="swiper" current="{{activeIndex}}" style="height: 100%">
        <block wx:for="{{contents}}" wx:key="*this">
            <swiper-item>
                <view style="height: 100%">
                    <text>{{contents[index]}}</text>
                </view>
            </swiper-item>
        </block>
    </swiper>
  </view>
</view>

以上代码是整个页面的布局， 可滑动的body我们用swiper组成，接下来是组成页面的数据滑动或者点击的方法，代码如下：

// pages/newMusic/index.js
Page({
  /**
  * 页面的初始数据
  */
  data: {
    activeIndex: 0,
    stv: {
      windowWidth: 0,
      lineWidth: 0,
      offset: 0,
      tStart: false
    },
    titles: ['文字一', '文字二', '文字三', '文字四'],
    contents: [
      "小时候，我穿着老爸的直筒袜，想象着我在大西部论坛球馆，投中制胜球的样子，我就深深明白一件事，我爱上了篮球，我爱你如此深以致贡献所有， 我的思想和身体， 我的精神和灵魂， 六岁开始， 我就深爱着你",
      "从未想到过结束，我只看到自己，在这条路上奔跑到尽头，所以我不停地跑，在 每个球馆我全力奔跑，为你争取每次机会，你要求我拼尽所有，所以我奉上真心，因为这里包含太多，我带着汗水和伤痛，不仅因为挑战",
      "而是你在召唤我，我为你奉献所有，因为你也赠予我全部，因为篮球，我感到是一个鲜活的人，你让6岁孩子实现湖人梦，为此我将永远爱你，但我不能再迷恋你，这个赛季是我仅能给，心还能继续承受打击，意志也还能被碾磨",
      "但身体知道是时候说再见，我已经准备让你离开，现在只是想让你知道，我们能永远珍藏着每段时光，无论是好的，还是坏的，我们已贡献给彼此，所有的全部，无论接下来我会做什么，我都会是那个孩子，穿着老爸直筒袜，瞄准角落里的垃圾桶"
      ]
    },

  /**
  * 生命周期函数--监听页面加载
  */
  onLoad: function (options) {},

  bodyChange: function(e) {
    console.log(e.target.dataset.value);
    this.setData({
      activeIndex: e.target.dataset.value
    });
  },
  swiper: function(e){
    this._updateSelectedPage(e.detail.current);
  },
  bodyscroll:function(e){
    console.log(e);
  },
  _updateSelectedPage(page) {
    let {titles, stv, activeIndex} = this.data;
    activeIndex = page;
    this.setData({activeIndex: activeIndex})
    stv.offset = stv.windowWidth*activeIndex;
    this.setData({stv: this.data.stv})
  },
  handlerTabTap(e) {
    this._updateSelectedPage(e.currentTarget.dataset.index);
  }
});

初识redux

Veröffentlicht am 2018-01-13

什么是Redux？

   Redux 是一个JavaScript App的可预测的状态管理器。

何时用Redux？

   曾经有人说过，如果你不知道是否需要Redux, 那么你就是不需要Redux.后来Redux的作者Dan Abramov又补充了一句只有遇到React实在解决不了的问题时候到你才需要使用Redux。简单讲，如果你的应用相对比较简单，视图层只从单一的数据源获取数据，操作相对比较简单，你就不需要使用Redux。拿React来作比，我们都知道，React的视图层的渲染依赖着每个组件的内部状态，当应用足够大了之后，就很难管理每个组件的状态，这时候，我们就需要有一个地方来存储整个应用的状态，此时，我们的redux就出场了.

设计思想

总结起来，两句话：

  • web应用是一个状态机器，视图与状态是一一对应的
  • 应用所有的状态，都保存在一个对象树里，随处可取

Redux 的安装

npm install redux --save

Redux的基本概念

Store

Store就是Redux管理整个web应用状态的地方，应用在任何地方可以从这棵状态树上取得数据，用于组件的渲染和计算
Redux提供createStore这个函数创建Store，整个应用只允许有一个Store。

import { createStore } from 'redux';
const store = createStore(fn);

State

前面我们介绍到，Store包含了整个web（当然此时每个组件内部也可同时维护自己的state）应用的状态数据，如果想得到某个时刻的状态，那么就可以从这棵状态树上获取自己的数据。这个时刻的数据，就叫做State,当前时刻的数据，可以通过store.getState()函数获取。

import { createStore } from 'redux';
const store = createStore(fn);
const state = store.getState();

在开发React的时候，我们知道，state对应view，跟视图层相关的state，如果state值相同，则渲染出来的UI相同。

Action

前面有提到过，State的变化，可能会导致View的变化。用户能接触到的是视图层，只能通过某种事件来触发State的改变，Action就是充当中间的信使，表示State要更新了
Action是一个对象，其中type属性是必须的,表述Action要做什么操作。

const action = {
  type: ADD_LIST,
  payload: 'Redux'
}

在上述代码中，要做的操作是ADD_LIST，所携带的信息payload的内容为Redux。从上述例子可以看出，Action描述的是将要发生的事，改变State唯一的途径就是通过触发Action。
